# sam-project

Travel demo SAM agent project. Lives in this repo; intended to be bind-mounted into the existing `sam-core-ee-1.171.0` container in `~/repo/ai-demo-for-solace-agent-mesh/`.

## Layout

- `configs/agents/` — agent YAML files. Disabled preset agents kept as `.disabled` so they are ignored by `sam run` but easy to re-enable.
- `configs/auth/`, `configs/platform.yaml`, `configs/webui_backend.yaml`, `configs/shared_config.yaml`, `configs/logging_config.yaml` — copied from the upstream `1.171.0-clean-start` profile.
- `tools/` — Python tool modules referenced by agent YAMLs via `component_module: tools.<module>`.
- `mock/` — JSON mock data consumed by Python tools (e.g. `mock/flights.json`).
- `db/` — SQLite databases generated by `../mockgen/` for `sam-sql-database`-backed agents.
- `logs/` — agent log files.

## Activating from the existing SAM stack

Run from `~/repo/ai-demo-for-solace-agent-mesh/`:

```shell
docker compose run --rm \
  -v /home/hhjau/repo/fy27-demothon/sam-project:/proj:rw \
  -p 8000:8000 \
  sam-core
```

Or edit `compose.yaml` `sam-core.volumes` to point at this directory instead of `./profiles/${SAM_PROFILE}/main`.

## Current agents

- OrchestratorAgent (preset, unchanged from `1.171.0-clean-start`).
- FlightsAgent (`configs/agents/flights_agent.yaml`): custom Python tool `lookup_flight` against `mock/flights.json`.
- WeatherAgent (`configs/agents/weather_agent.yaml`): sam-sql-database-backed agent against `db/weather.db`.
- ItineraryAgent (`configs/agents/itinerary_agent.yaml`): sam-sql-database-backed agent against `db/itinerary.db`.

The sam-sql-database agents reference the plugin module directly in YAML (`agent_init_function.module: sam_sql_database.lifecycle`, tool `component_module: sam_sql_database.tools`). The `sam plugin add ... --plugin sam-sql-database` CLI is intentionally avoided — see https://repo.jbetalabs.com/solace/fy27-demothon for the known bug context. The plugin module is expected to be importable inside the running SAM container; demo01-insurance uses the same pattern in this stack.

Required env vars (already in `configs/agent.env`):

- `A_WEATHER_DB_TYPE=sqlite`, `A_WEATHER_DB_NAME=db/weather.db`
- `A_ITINERARY_DB_TYPE=sqlite`, `A_ITINERARY_DB_NAME=db/itinerary.db`

Regenerate the DBs from `../mockgen/`:

```shell
cd ../mockgen && uv run python generate.py all
```
